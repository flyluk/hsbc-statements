Context: I have an Excel workbook with transaction data in a sheet called "All Transactions".

Columns:
A Account
B Date
C Transaction Details
D Deposit
E Withdrawal
F Amount
G Balance
H Currency
I Source File

Create / rebuild a "Summary" sheet with 3 formula-based monthly summary tables.

General requirements:
- Add hidden helper columns in "All Transactions":
  J Category
  K Month = YYYY-MM
  L IsCC
  M IsMortgage
  N Mortgage Principal
  O Mortgage Interest
- Use live formulas, primarily SUMIFS, referencing helper columns.
- Months should be one group per month present in the data.
- Each month group should have In / Out / Net columns.
- Add Total In / Total Out / Total Net columns.
- Format with bold headers, borders, thousands separator, yellow total rows.
- Use blue headers for credit cards, green for bank accounts, purple for mortgage.

Table 1: Credit Card Transactions Summary
- Include categories:
  Adjustment, Apple, DCC Fees, Dining, Entertainment, Groceries, Home/Shopping, Instalment, Instalment Fee, Medical/Dental, Mobile, Software/Subscription, Tax Payment, Transfer/Other
- IsCC = 1 when Account contains Visa, Credit Card, CLEAR CARD, or CARD.
- Exclude card payment transactions such as THANK YOU, PAID BY AUTOPAY, PAYMENT.
- Summarize by Category + Month.

Table 2: Bank Account Transactions Summary
- Include categories:
  Opening Balance, Salary, Interest, Claims Payout, Alipay, Car Loan, Credit Payment, Entertainment, Government, Home/Shopping, Insurance, Investment, Management Fee, Octopus, Rent, Standing Order, Transfer/Other, Utilities
- IsCC = 0 and IsMortgage = 0.
- Map "CITIBANK EUROPE PLC" to Claims Payout.
- Map Alipay transfers containing name + A852 reference to Alipay.
- Opening Balance should use Amount column F positive values because Deposit / Withdrawal may be blank.
- Exclude all mortgage payments from mortgage statement in bank summary.

Table 3: Mortgage Transactions Summary
- Include only:
  Principal
  Interest
- Use IsMortgage = 1.
- Identify mortgage rows by Account containing Mortgage or Transaction Details containing mortgage loan references (e.g. loan number / 支付貸款).
- Parse principal and interest from mortgage account Transaction Details when available.
- If principal / interest are missing or zero, infer principal from mortgage balance change.
- For bank-only mortgage payments with no matching mortgage account row on the same date, allocate payment using the latest known principal / interest ratio.
- Avoid double-counting when both bank payment row and mortgage account row exist on the same date.
